SF Knowledge Hub — Firestore data topology

Generic reference. "course" stands in for any tool track (the pattern is identical across all courses).

1. Certification data

How a course's pages write progress and exam data, and which admin readers consume it. Each lifecycle has a legacy collection (written, not read by admin) and a v2 collection (read by admin); the dual-write keeps both populated, and backfills bridge legacy data into the readable side.

2. Proctoring & scheduling data

How learner, proctor, and admin actions write slots, sessions, snapshots, and reschedule data. The three actors are the same hub pages rendered in different roles. This subsystem shares only the users and exams catalog collections with view 1.

Not drawn: the courses catalog and adminConfig, which are read almost everywhere. courses/{autoId} is the catalog — its doc id is the courseId / linkedCourseId that keys courseExamResults and the exam definitions. Firestore rules are not shown.
